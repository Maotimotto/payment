# 个人记账应用 PRD（v1.0）

> 状态：已锁定 | 类型：纯本地 / 隐私优先 / LLM 可选 的个人记账 Web 应用

---

## 1. 产品定位与目标用户

- **一句话定位**：一个纯本地、隐私优先、可选接入大模型的小而美个人记账工具，核心能力是「多平台账单聚合 + 智能解析 + 跨平台去重 + 标签化可视化」。
- **目标用户**：有多平台消费（微信 / 支付宝 / 银行卡）、希望把账目统一管理、且关注隐私（不愿数据上云）的个人用户。
- **核心差异点**：
  1. LLM 辅助解析任意格式账单；
  2. 跨平台去重；
  3. 数据纯本地不上云。

---

## 2. 核心原则（设计红线）

1. **纯本地**：所有数据存于浏览器 IndexedDB，无后端、无账号、不上云。
2. **LLM 可选**：未接入也能用全部核心功能（手动录入 + CSV 规则解析）；接入后解锁截图识别 + 任意格式智能解析。
3. **克制**：能用规则做的不用 LLM，能不做的功能先不做。

---

## 3. 功能列表（按优先级）

### P0 — MVP 必做

| 模块 | 功能 |
|---|---|
| 账本 | 创建 / 切换 / 重命名 / 删除多账本 |
| 账单导入 | CSV 上传（微信 / 支付宝 / 招商银行 内置规则解析） |
| 智能解析 | 截图识别（需 LLM）；任意 CSV 经 LLM 分批解析（50 条/批，规则提结构化字段、LLM 只做分类打标签）；失败降级本地规则 |
| 手动录入 | 单笔录入（金额 / 时间 / 分类 / 支付方式 / 备注） |
| 去重 | 双窗口：普通消费金额相同 + 时间差 ≤10s；还款/转账类 ≤24h → 标记"疑似重复" → 用户一键确认合并 |
| 组合支付 | 拆为多条 + 视觉分组 + "组合支付"标注，不去重不合并 |
| 标签 | 多级标签；预设 + 用户增删 + 自定义；单主标签 + 备注；含"信用卡还款"等特殊标签 |
| 收支处理 | 收入与支出对称解析，资金搬运类（还款/转账/赎回）不计入收支统计 |
| 条目操作 | 查看 / 编辑 / 删除 / 批量删除 / 手动改标签 |
| 搜索筛选 | 按金额 / 商户 / 标签 / 时间筛选 |
| Dashboard P0 | 顶部卡片（当月收入/支出/结余/环比）+ 分类饼图 + 近 6 月趋势折线 |
| 导出 | 导出 Excel |
| LLM 配置 | API Key + Base URL + 模型（预设+自定义）+ 测试连接；存 IndexedDB 不加密并明确告知 |
| 体验 | 暗色模式 |

### P1 — 紧随其后

- Dashboard 其余图表：单笔支出 Top10、各账本对比柱状、消费日历热力图、支付渠道占比
- 扩展更多银行/平台的内置规则模板

### P2 / 后续（明确不在当前范围）

- 邮箱账单抓取、预算告警、多用户/家庭共享（需云端）、多币种、原始文件留底

## 4. 核心流程

```
账单输入（CSV / 截图 / 手动）
   │
   ├─ CSV：先匹配内置规则模板 ──成功──┐
   │         └─不匹配且有LLM─► LLM分批解析 ┤
   │         └─不匹配且无LLM─► 提示手动录入 │
   ├─ 截图（需LLM）─► LLM多模态解析 ───────┤
   └─ 手动录入 ────────────────────────────┤
                                            ▼
                                   结构化账单条目（草稿）
                                            ▼
                          去重检测（金额=、双窗口时间）
                                            ▼
                      标记"疑似重复" → 用户确认合并/保留
                                            ▼
                          分类打标签（LLM 或 规则/默认）
                                            ▼
                                  入库（IndexedDB）
                                            ▼
                              Dashboard 可视化 + 明细查询
```

---

## 5. 数据模型

- **Ledger（账本）**：id、名称、创建时间、币种（默认 CNY）
- **Transaction（账单条目）**：id、账本id、金额、方向（收/支）、发生时间、商户/描述、来源平台、支付方式、主标签id、备注、`是否计入收支统计`、`组合支付组id（可空）`、`去重状态`、原始数据快照（文本）
- **Tag（标签）**：id、名称、父标签id（支持多级）、是否预设、是否特殊类型（如信用卡还款）
- **PaymentMatchGroup（去重/组合关联组）**：id、类型（去重合并 / 组合支付）、关联条目id列表
- **LLMConfig**：apiKey、baseURL、modelName、是否启用
- **ParseTemplate（内置规则模板）**：平台名、字段映射规则

---

## 6. 页面清单

1. **Dashboard（首页）**：卡片 + 图表 + 账本切换
2. **导入页**：上传 CSV / 上传截图 / 手动录入入口 + 解析进度
3. **去重确认页/弹窗**：疑似重复列表，逐组确认
4. **账单明细页**：列表 + 搜索筛选 + 批量操作 + 编辑改标签
5. **标签管理页**：多级标签树增删改
6. **设置页**：LLM 配置（含测试连接）+ 暗色模式 + 导出 + 数据管理

---

## 7. 去重规则定稿

| 场景类型 | 匹配条件 | 处理 |
|---|---|---|
| 普通消费跨平台双记（S1） | 金额相同 + 时间差 ≤10s | 标记疑似重复 → 用户确认合并 |
| 还款/转账类（S2，花呗还款、信用卡还款、账户互转） | 金额相同 + 时间差 ≤24h | 标记疑似重复 → 用户确认合并 |
| 组合支付（S3） | 同订单多笔支付 | 不去重、不合并，分组展示 |
| 信用卡还款（S4） | —— | 打"信用卡还款"标签，不计入收支统计 |

## 8. 关键风险与应对

| # | 风险 | 应对 |
|---|---|---|
| R1 | 去重窗口漏判 | 双窗口（普通 ≤10s / 还款转账 ≤24h），已采纳 |
| R2 | 截图识别准确率 | 解析后强制进入用户确认环节 |
| R3 | LLM 分类不稳定 | 建立"商户→标签"本地记忆，命中直接套用，省 token 且更稳 |
| R4 | 招行 CSV 格式变动 | 模板版本化，失效时降级 LLM/手动 |
| R5 | 多级标签饼图口径 | 默认按一级聚合，可下钻 |

---

## 9. MVP 验收标准

> 用户无需联网账号，能上传微信/支付宝/招行 CSV（或截图，或手动录入），系统自动解析打标签、识别疑似重复并由用户确认，最终在 Dashboard 看到当月收支、分类饼图和趋势，并能导出 Excel。

## 10. 技术选型（开发阶段确定）

- **构建/框架**：Vite + React + TypeScript
- **样式**：Tailwind CSS（内置暗色模式）
- **本地存储**：IndexedDB（Dexie.js 封装）
- **状态管理**：Zustand
- **图表**：Recharts
- **CSV 解析**：PapaParse
- **Excel 导出**：SheetJS（xlsx）
- **LLM 调用**：fetch 直连用户配置的 OpenAI 兼容接口（含多模态截图）
